# LiveKit Voice Agent — Swades AI Hackathon

A conversational voice agent with live monitoring and warm transfer built with LiveKit, OpenAI, Deepgram, and Twilio.

## Features
- 🤖 AI voice agent (Clara) for appointment booking
- 📅 Real-time appointment booking with SQLite
- 🖥️ Live monitoring dashboard with transcript
- 🙋 Watcher takeover control
- 📞 Warm transfer to human agent via Twilio
- 📋 Post-call AI summary

## Tech Stack
| Layer | Tech |
|---|---|
| Backend | Python, FastAPI, LiveKit Agents SDK |
| Frontend | Next.js, Tailwind CSS, LiveKit Components |
| Voice AI | OpenAI GPT-4o-mini, Deepgram STT/TTS |
| Database | SQLite (aiosqlite) |
| Telephony | Twilio, LiveKit SIP |

## Setup

### 1. Clone the repo
```bash
git clone https://github.com/yourusername/livekit-hackathon.git
cd livekit-hackathon
```

### 2. Backend Setup
```bash
cd backend
pip install -r requirements.txt
```

Fill in your `.env` file:
```env
LIVEKIT_URL=wss://your-project.livekit.cloud
LIVEKIT_API_KEY=your_livekit_api_key
LIVEKIT_API_SECRET=your_livekit_api_secret
OPENAI_API_KEY=your_openai_api_key
DEEPGRAM_API_KEY=your_deepgram_api_key
TWILIO_ACCOUNT_SID=your_twilio_account_sid
TWILIO_AUTH_TOKEN=your_twilio_auth_token
TWILIO_PHONE_NUMBER=+1xxxxxxxxxx
TRANSFER_PHONE_NUMBER=+1xxxxxxxxxx
```

### 3. Frontend Setup
```bash
cd frontend
npm install
```

Fill in `frontend/.env.local`:
```env
NEXT_PUBLIC_LIVEKIT_URL=wss://your-project.livekit.cloud
NEXT_PUBLIC_SERVER_URL=http://localhost:8000
```

## Running the App

### Terminal 1 — FastAPI Backend
```bash
cd backend
python server.py
```

### Terminal 2 — LiveKit Voice Agent
```bash
cd backend
python agent.py dev
```

### Terminal 3 — Next.js Frontend
```bash
cd frontend
npm run dev
```

Then open http://localhost:3000

## How It Works

### Booking Flow
1. Caller visits http://localhost:3000 and enters their name
2. Joins a LiveKit room and speaks with Clara (AI agent)
3. Clara collects name, reason, date, time, and contact
4. Clara checks availability in SQLite and confirms booking

### Monitoring Flow
1. Watcher opens the Dashboard
2. Enters the room name from an active call
3. Sees live transcript and agent state in real time
4. Can take over the call at any time

### Warm Transfer Flow
1. Caller says "billing", "complaint", or "speak to a human"
2. Clara detects intent and calls the human agent via Twilio
3. Clara speaks a summary and asks human to accept or decline
4. If accepted → caller connected to human; if declined → Clara returns

### Post-Call Summary
- Generated by GPT-4o-mini after call ends
- Shows duration, messages, appointment details, and AI summary

## API Endpoints
| Method | Endpoint | Description |
|---|---|---|
| GET | /token | Get caller token |
| GET | /monitor-token | Get monitor token |
| GET | /takeover-token | Get takeover token |
| GET | /call-state/{room} | Get live call state |
| POST | /end-call | End call + generate summary |
| POST | /update-state | Update agent state |
| POST | /save-transcript | Save transcript entry |
| WS | /ws/monitor/{room} | Live monitoring websocket |
